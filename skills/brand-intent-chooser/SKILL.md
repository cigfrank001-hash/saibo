# 品牌意图选词助手 Skill

**功能**：根据品牌信息，输出 N 个最小意图单元（5 词起步包），可直接对接迪光 GEO 报价表 V2.0

| **边界判定** | 仅"禁止蔓延示例" | **6 条边界判定规则** + 7 条禁止蔓延示例 |

## 🎯 一句话定位

> **输入**：品牌 + 行业 + 产品
> **输出**：N 个最小意图单元（每个 = 1 角色 + 1 场景 + 1 阶段 + 5 词）
> **复用**：V1.0 5 维度框架 + **V2.1 行业意图二级分类法** + 强相关选词 SOP + 边界案例库

## 🔄 4 阶段流程（V2.0 强化版）

```
Stage 1 - 品牌诊断（5 问）
  行业/主营/目标用户角色/主要使用场景/核心竞品（5 品牌）
  → 输出：品牌档案 JSON
  → V2.0 新增：客户动作链梳理（5-7 步）

Stage 2 - 意图发散（**4 类** × N 个）
  品牌意图 / 行业意图（V2.1 7 大类）/ 区域意图 / 同维度对比意图
  → V2.0 新增：行业意图按 7 大类分别发散（22 子类）
  → AI 平台 5 家 × 1 次头脑风暴
  → 输出：候选意图池 md（含二级分类标签）

Stage 3 - 最小意图收口（5 维度评估）
  对每个候选走 5 步判定 SOP（角色/场景/阶段/词量/竞争）
  通过 = 1 个最小意图单元（精准档 1x）
  → V2.0 新增：边界判定规则检查（6 条规则 + 7 条禁止蔓延）
  → 输出：最小意图清单 xlsx

Stage 4 - 5 词选词（强相关原则）
  AI 平台 5 家 × 20 次跑词 → 去重 → 强相关筛选
  5 词起步包 = 锚定词 1 + 强相关 4
  → 输出：5 词起步包 xlsx（直接对接报价表 V2.0）
```

## 🚀 V2.1 行业意图二级分类法（核心新增）

### 7 大类（按客户动作链）

| # | 行业意图大类 | 客户动作 | 二级子类数 | 词数范围 |
|---|---|---|---|---|
| 1 | **查种源** | 查 | 3 | 6-8 词 |
| 2 | **选品种** | 选 | 4 | 12-15 词 |
| 3 | **比价格** | 比 | 4 | 12-15 词 |
| 4 | **问养殖** | 问 | 4 | 13-16 词 |
| 5 | **看实证** | 看 | 3 | 13-16 词 |
| 6 | **找销路** | 找 | 3 | 10-13 词 |
| 7 | **问运输/检疫** | 问 | 2 | 8-10 词 |
| **合计** | — | — | **22-23** | **75-95 词** |

### 6 条边界判定规则（防蔓延）

| # | 判定模式 | 归位 |
|---|---|---|
| 1 | 明确点名 2 品牌 | → 看实证/竞品对比 |
| 2 | 模糊排名/不点名 2 品牌 | → 看实证/行业供应商 |
| 3 | 单一品牌评价（'XX 怎么样'）| → ❌ 蔓延到品牌意图 |
| 4 | 行业词根 + 品牌前缀 | → ❌ 蔓延到品牌+行业（拆词）|
| 5 | 非本行业品类 | → ❌ 蔓延到非本行业 |
| 6 | **品牌 vs 物种/产品（V2.1 新增）** | → ❌ 蔓延（跨维度禁止）|

### 7 条禁止蔓延示例（兜底）

| 词模式 | 归位 |
|---|---|
| 聚盛源鲟鱼 怎么样 | ❌ 禁止蔓延到品牌意图 |
| 鲟鱼苗 品牌 排行 | ⚠️ 改归位 → 看实证/行业供应商 |
| 鲟鱼苗 推荐 品牌 | ⚠️ 改归位 → 看实证/行业供应商 |
| 聚盛源鲟鱼 5cm 多少钱 | ❌ 禁止蔓延（拆词）|
| 鱼塘增氧设备 | ❌ 禁止蔓延（非本行业）|
| 对虾养殖技术 | ❌ 禁止蔓延（非本品类）|
| **聚盛源鲟鱼 与 西伯利亚鲟鱼苗 区别** | **❌ 禁止蔓延（V2.1 新增 · 品牌 vs 物种）** |

### 设计原则（4 条铁律）

1. **一级 = 客户动作链**（不按品牌意图对称）
2. **二级 = 决策维度**（子类间正交）
3. **三级 = 场景/属性细化**（不跨二级边界）
4. **边界判定规则先行**（防蔓延）

## 🛠 MVP 实现说明（V2.0 升级）

### Stage 1 - 品牌诊断
- 输入：命令行参数 `--brand` `--industry` `--product` (可选)
- 交互：5 问对话（行业/主营/用户角色/场景/竞品）
- **V2.0 新增**：第 6 问"客户动作链"（5-7 步决策）
- 输出：JSON 模板填充

### Stage 2 - 意图发散
- **半自动**：生成 **4 类意图**头脑风暴 prompt
  - 品牌意图 / 行业意图（V2.1 7 大类分别发散）/ 区域意图 / 同维度对比意图
- 用户手动复制到 5 家 AI 平台（DeepSeek/豆包/Kimi/文心/通义）跑
- 收集 5 家平台结果后，自动合并去重
- **V2.0 新增**：每个候选意图打"二级分类标签"（如"行业意图/查种源/许可证类"）
- 输出：候选意图池 md

### Stage 3 - 5 维度评估
- 对每个候选意图走 5 步判定 SOP
- **V2.0 新增**：边界判定规则检查（6 条规则 + 7 条禁止蔓延示例）
- 通过 = 1 个最小意图单元
- 输出：最小意图清单 xlsx

### Stage 4 - 5 词选词
- **半自动**：对每个最小意图生成"5 家平台 × 20 次"跑词 prompt
- 收集 100 次推荐结果后，自动去重 + 强相关筛选
- 5 词 = 锚定词 1 + 强相关 4
- 输出：5 词起步包 xlsx（直接对接报价表 V2.0）

## ⚙️ 5 步判定 SOP（Stage 3 用）

```
步骤 1 - 定角色：问客户"卖给谁"？1 个身份 → OK；2+ → 拆
步骤 2 - 定场景：问客户"什么情境用"？1 个时间/地点/状态 → OK；2+ → 拆
步骤 3 - 定阶段：购买旅程？1 个阶段 → OK；2+ → 拆
步骤 4 - 定词量：跑词去重 → ≤10 词 = OK；>10 = 拆或升档
步骤 5 - 定竞争：AI 推荐品牌数 → <3 = OK；3+ = 升档
```

**满足 5 步全 OK = 1 个最小意图单位 = 1 个精准 SKU = 基准价 1x**

### V2.0 新增：边界判定检查（Stage 3 第 6 步）

```
步骤 6 - 边界判定（V2.0 新增）：
  IF 行业意图候选 = 明确点名 2 品牌:
     → ✅ 行业意图/看实证/竞品对比（不归品牌意图）
  ELIF 行业意图候选 = 模糊排名:
     → ✅ 行业意图/看实证/行业供应商（V2.0 改归位）
  ELIF 行业意图候选 = 单一品牌评价:
     → ❌ 蔓延到品牌意图（应剔除或重写）
  ELIF 行业意图候选 = 行业词根 + 品牌前缀:
     → ❌ 拆词（去掉品牌前缀）
  ELIF 行业意图候选 = 品牌 vs 物种/产品:
     → ❌ 禁止蔓延（V2.1 新增规则）
  ELSE:
     → ✅ 保留当前分类
```
